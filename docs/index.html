DLP Backend - API Documentation

Quick links to the per-service OpenAPI specs and Swagger UI pages (served statically from each service's doc/ folder):

Configuration Service

Spec: openapi.yaml

Swagger UI: swagger-ui.html

Run example: create a DataType

curl -X POST "http://localhost:8080/api/datatypes" -H "Content-Type: application/json" -d '{"name":"Sensitive Keywords","type":"KEYWORDS","content":["password","secret"]}'

Policy Service

Spec: openapi.yaml

Swagger UI: swagger-ui.html

Run example: list datasets for a tenant

curl "http://localhost:8081/api/policies/tenant/3fa85f64-5717-4562-b3fc-2c963f66afa6"

Scanner Service

Spec: openapi.yaml

Swagger UI: swagger-ui.html

Run example: trigger a scan

curl -X POST "http://localhost:8082/api/scans" -H "Content-Type: application/json" -d '{"tenantId":"3fa85f64-5717-4562-b3fc-2c963f66afa6","dsId":"","input":"some text"}'

Notes:

  • To preview the Swagger UI pages directly, serve the files from the service's doc folder. For example: python -m http.server 8083 --directory services/configuration-service/doc and open http://localhost:8083/swagger-ui.html.
  • When running services in Docker Compose the doc files are available inside the container at /app/doc (if you copy them there). Alternatively, serve them locally as shown above.
